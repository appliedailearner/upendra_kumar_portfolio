Rate Optimization: Reservations & Savings Plans

04

Azure Reservations

Commit to 1-year or 3-year plans for predictable workloads to secure up to 72% discount over pay-as-you-go rates.

  • Virtual Machines: Reserved 80+ VM instances (D-series, E-series) for production workloads
  • SQL Database: Reserved capacity for mission-critical databases
  • Azure Synapse: Reserved compute for data warehouse workloads

Azure Savings Plan

Flexible pricing model with commitment to consistent usage ($/hour) across any eligible services.

  • Compute Savings Plan: Applied to VMs, App Service, Container Instances
  • Flexibility: Automatically applies to any region, instance size, or OS

Commitment Strategy

1-Year Reservations -40%
3-Year Reservations -62%
Savings Plan (Compute) -65%
Coverage Analysis

Used Azure Advisor to analyze 7, 30, and 60-day lookback periods to identify optimal reservation coverage.

$420k/year

Reservations + Savings Plan Savings

Usage Optimization: Rightsizing

05

Azure Advisor Recommendations

Leveraged Azure Advisor to identify over-provisioned resources and optimize VM SKUs based on actual utilization patterns.

  • VM Rightsizing: Downsized 45+ VMs from D16s to D8s (50% cost reduction)
  • Storage Optimization: Moved infrequently accessed data to Cool/Archive tiers
  • Database SKUs: Optimized SQL Database DTU/vCore allocation based on workload patterns
Utilization Metrics

Analyzed CPU, memory, and disk I/O over 30-day periods to identify candidates for downsizing.

Optimization Impact

Compute Reduction

25% reduction in total compute costs through VM rightsizing

Storage Tiering

18% storage cost savings via Cool/Archive tier migration

Database Optimization

12% SQL Database cost reduction via DTU optimization

$100k/year

Rightsizing Savings | 45+ VMs Optimized

Usage Optimization: Idle Resource Cleanup

06

Automated Detection

Implemented Azure Policy and automation to identify and remediate idle resources consuming budget without delivering value.

  • Orphaned Disks: Detected and deleted 120+ unattached managed disks
  • Unused Public IPs: Removed 35+ unassociated public IP addresses
  • Stopped VMs: Identified VMs stopped for 30+ days and deallocated them
  • Dev/Test Schedules: Auto-shutdown policies for non-production environments (7 PM - 7 AM)
Automation

Azure Automation runbooks execute weekly scans and send cleanup recommendations to resource owners.
