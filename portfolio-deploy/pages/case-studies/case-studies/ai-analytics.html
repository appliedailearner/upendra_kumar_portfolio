The Infrastructure Foundation

1. Secure Connectivity

Architected a Hub-Spoke model where the Data Landing Zone (Spoke) was isolated from direct internet access. Ingestion from AWS S3 and On-Premises SQL was routed via VPN Gateway and ExpressRoute, ensuring data never traversed the public internet.

2. Identity & Governance

Implemented a Persona-Based Access Model using Entra ID Security Groups:

  • PlatformAdmins: Infrastructure CI/CD access.
  • DataEngineers: Access to Data Factory and Storage Account (Contributor).
  • DataScientists: Access to ML Workspace and Compute Targets.

Multi-Cloud & Vendor Integration

A key challenge was integrating existing 3rd party platforms. I led the technical validation for:

  • Snowflake: Configured Private Link connectivity to Azure VNet to avoid public endpoint exposure.
  • Databricks: Managed VNet injection planning to ensure Databricks clusters operated within our IP address space constraints.

Business Outcomes

Revenue Growth: The stable platform allowed the Data Science team to ship a "Next Best Action" recommendation engine in 6 weeks, driving a 15% revenue uplift.

Security Compliance: Passed all InfoSec penetration tests due to the strict Private Endpoint implementation and Identity boundaries.
