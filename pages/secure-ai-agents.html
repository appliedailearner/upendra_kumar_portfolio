Flagship Architecture

Secure AI Agents on Azure

A reference architecture for deploying production-grade Geneative AI agents within a secure, governed enterprise perimeter.

Figure 1: High-Level Architecture for Secure Enterprise AI Agents

1. Network Security (Private Perimeter)

  • No Public Access: All AI services (OpenAI, Search, Cosmos DB) accessible only via Private Endpoints.
  • Hub-Spoke Topology: Traffic Inspection via Azure Firewall in the Hub before reaching the AI Spoke.
  • VNET Integration: App Services and Function Apps strictly VNET-integrated.

2. Identity & Access (Zero Trust)

  • Managed Identities: No secrets in code. Use System-Assigned MI for all service-to-service auth.
  • Entra ID RBAC: Granular "Cognitive Services User" roles; no broad contributor access.
  • Key Vault: Minimal usage (only for 3rd party API keys), backed by RBAC.

3. Operational Excellence (GenAIOps)

  • Evaluation Framework: Automated testing of prompts using prompt flow in CI/CD.
  • Content Safety: Azure AI Content Safety filters enabled and tuned for Jailbreak detection.
  • Cost Guardrails: Token usage limits and budget alerts configured per department.

4. Integration Patterns

  • Orchestration: Semantic Kernel or LangChain hosted on Container Apps.
  • RAG Pattern: Hybrid Search (Vector + Keyword) using Azure AI Search for grounding.
  • API Gateway: APIM as the front door for throttling and consumption tracking.
